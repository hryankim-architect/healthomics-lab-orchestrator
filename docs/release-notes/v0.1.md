# `healthomics-lab-orchestrator` v0.1

*Public reference subsetted to chr22; the full GRCh38 run exceeds the size target for this demo.*

First public release of the substrate-aware RNA-seq orchestration demo.
Runs the Nextflow DSL2 + SHA-256-chained NDJSON ledger + MLflow tracking
stack on a chr22-subset reference. End-to-end runtime is ~45 seconds on a
single Mac/Linux box. No GPU, no Docker, no cloud credentials required.

---

## The capability

A four-process Nextflow DSL2 pipeline (FastQC -> HISAT2 -> featureCounts ->
MultiQC) wrapped in a Python orchestrator that brackets the run with a
hash-chained audit ledger and MLflow tracking. Every Nextflow process emits
its own start/end audit entry through a CLI substrate hook, so the resulting
chain spans both the outer orchestrator and every inner step as a single
linear history.

```
pipeline_start                                              <- outer bracket
   |
   +---- FASTQC x 3        (start/end + per-sample audit)   <- inner Nextflow
   +---- HISAT2_ALIGN x 3  (start/end + align_rate_pct)
   +---- FEATURECOUNTS x 3 (start/end + assigned_pct)
   +---- MULTIQC x 1       (start/end + aggregate)
   |
pipeline_end                                                <- outer bracket
```

**Result**: a 22-entry NDJSON ledger with SHA-256 chaining that any substrate
consumer (e.g. `lab_semantic_check.py`) can read to reconstruct
the entire run.

## Real-data climax

End-to-end smoke on chi-mac-p, 3 paired-end samples x 1M reads each:

| Metric | Value |
|---|---|
| Wall-clock (fresh run, all 10 tasks) | **43.8 seconds** |
| Wall-clock (`-resume`, all cached) | ~8 seconds |
| HISAT2 alignment rate per sample | 3.69% / 3.50% / 3.46% |
| featureCounts assigned per sample | ~3.0% (chr22 expressed-transcript share) |
| MultiQC HTML | 2.4 MB, 12 reports aggregated |
| Audit chain entries | **22** (1 + 20 + 1, all start/end paired) |
| Audit chain validity | `ok=True`, every `prev_hash` verifies |
| Per-process MLflow metrics | `align_rate_pct`, `assigned_pct`, `assigned_reads` |
| Aggregate MLflow metrics | `nextflow_wall_clock_seconds`, `audit_chain_entries`, `multiqc_report_bytes` |

The 3-4% alignment rate is the *correct* number for a chr22-only reference
(chr22 is ~1.6% of GRCh38 by length, ~3% by expressed-transcript content).
That is the engineering proof, not a biology weakness, see the
"Scope" section in README and `docs/what-is-out-of-scope.md` for
the full reasoning.

## macOS-Nextflow integration lessons (the substrate war stories)

Hour-3 real-data smoke surfaced six cascading lessons specific to running
Nextflow on macOS in a conda + uv environment. All are codified in
`docs/tooling-versions.md` with the relevant fixes landed in
`nextflow.config` (`env { PATH = ... }`) and `scripts/run_lab.sh`:

| ID | Symptom | Fix |
|---|---|---|
| L-phi | MultiQC libtiff soname mismatch in base conda env | dedicated `multiqc` conda env + `conda run -n multiqc` |
| L-psi | Nextflow rejects conda Java 25 | `conda deactivate; unset JAVA_HOME JAVA_CMD` -> system Java 19 |
| L-omega | `python: command not found` after L-psi fix | prepend `${projectDir}/.venv/bin` to `env.PATH` |
| L-alpha2 | `fastqc: command not found` after L-omega fix | append `${HOME}/miniconda3/bin` to `env.PATH` |
| L-beta2 | Nextflow `-resume` cache key ignores env-var changes | `rm -rf work/` + fresh run, or `scripts/run_lab.sh --fresh` |
| L-chi | `zsh: event not found:` in commit messages with `!` | `git commit -F file` + quoted heredoc |

These are *not* mistakes the next contributor needs to rediscover. They are
documented to make the lab repo behave the same way on every macOS
workstation that runs `scripts/run_lab.sh`.

## Quickstart

```bash
gh repo clone hryankim-architect/healthomics-lab-orchestrator
cd healthomics-lab-orchestrator

make install        # uv sync --extra dev
make data           # downloads chr22 ref + 3 SRA samples (~110 MB)
scripts/run_lab.sh  # macOS-hardened end-to-end run (~45 s)

# View results
open results/multiqc/multiqc_report.html
cat audit/local-demo.ndjson | head -3
```

## Lineage

Built on `bioinformatics-repo-scaffold-template`, the shared scaffold that
other repos in this portfolio also inherit. Substrate modules (`audit.py`,
`tracking.py`, `canary.py`, `process_hooks.py`) are designed to be
copy-and-edit, not pip-installed, so each repo can diverge as needed
without coordinating releases.

The analytical-method side (TP53 hotspot tiering + HRD severity score on
TCGA-LAML, Cox HR 8.39 / p=0.024) lives in a separate repo in this
portfolio.

## What v0.1 deliberately does not do

- No full reference genome (chr22 only)
- No differential expression analysis (orchestration, not analysis)
- No container engine (host tools via bioconda + brew)
- No multi-organism support
- No production HA / RBAC / multi-tenant
- No cloud-cost optimization
- No real-time / streaming inputs
- No alternative aligners (HISAT2 only)

See `docs/what-is-out-of-scope.md` for the full scope-boundary ledger with
reasoning for each item.
